The same task, before and after translation.

On this set the models read patient phrasing perfectly. The losses appear after the translation step.

The figures elsewhere trace circuits inside one small model. This page asks a different question of production-scale models: can a Claude model pull the clinical concepts out of a sentence, and does translating it first help or hurt? Each model is scored on the same items, so the two percentages are directly comparable.

Results

Model Patient → clinician phrasing Run cost

Content was lost at the translation step itself.

This reverses the gemma story. gemma-2-2b, a 2-billion-parameter model, loses the clinical continuation on patient phrasing. These production models did not — every one scored 100% on the untouched patient wording. The losses came from translating, the same regression the engine flags, and the reason the Translation fix can’t be assumed safe without checking its output.

Read this with its limits

The item set is small, so single items move the percentages a lot; read the numbers as a demonstration of the method and the direction of the effect, not a ranking of models. Scoring matches against expected clinical terms, so a valid synonym the list lacks would count as a miss. Runs hit a live API on a specific date and can differ on re-run. See the broader limitations for what this evidence does and doesn’t show.
